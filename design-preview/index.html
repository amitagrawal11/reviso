Accessibility

Cheatsheet + semantic HTML reference. One file because they're inseparable.

Tier 1: Non-negotiables (do these first)

1. Semantic HTML before ARIA

"The first rule of ARIA is don't use ARIA." Native elements come with built-in keyboard support, focus management, and screen reader semantics. 
<!-- ✓ -->
<button onClick={...}>Submit</button>
<a href="/profile">Profile</a>

<!-- ✗ -->
<div onClick={...} role="button">Submit</div>

2. Heading hierarchy

One <h1> per page, then descend without skipping levels.

3. Page landmarks

Use <header>, <nav>, <main>, <aside>, <footer> so assistive tech can jump between regions.
